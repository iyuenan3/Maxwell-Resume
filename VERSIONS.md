# 简历版本管理规则

## 📋 核心规则

### 1. 文件命名
- **不带时间戳** = 最新版本（工作文件）
  - `resume-base.md` - 基础版本
  - `openclaw-integration-engineer.md` - OpenClaw 岗位版
- **带时间戳** = 历史备份（仅保留一份）
  - `resume-base-2026-03-11-13-27.md` - 备份版本
  - 格式：`YYYY-MM-DD-HH-MM`

### 2. 文件类型
- **只保留 Markdown (.md)**
- ❌ 不生成 HTML
- ❌ 不生成 PDF

### 3. 版本管理
- **最新版本**: 无时间戳的 MD 文件（直接编辑）
- **历史备份**: 每次重大修改后复制一份带时间戳的备份
- **仅保留一份备份**: 删除旧备份，只保留最新版备份

### 4. 简历结构（固定 6 大板块）

所有简历版本必须按以下顺序组织：

1. **个人定位**
   - 职业定位（如：技术负责人 / AI 工程化专家）
   - 核心优势概述

2. **核心技能**
   - 按类别列出技术栈
   - 包含量化成果（如：成本优化 31%）

3. **工作经历**
   - 按时间倒序排列
   - 每段经历包含：公司 | 职位 | 时间
   - 核心成果 + 详细职责

4. **个人项目**
   - 开源项目
   - 个人作品
   - 包含 GitHub 链接

5. **教育经历**
   - 学校 | 专业 | 学历 | 时间
   - 主修课程
   - 校园活动

6. **个人专利**
   - 专利列表
   - 包含专利号

### 5. URL 格式
- ✅ **正确**: `https://github.com/iyuenan3/k8s-om`（完整 URL）
- ❌ **错误**: `[k8s-om](github.com/iyuenan3/k8s-om)`（Markdown 链接）
- **原因**: 保持简洁，便于阅读和复制

## 🎯 工作流程

### 修改简历时

1. **编辑工作文件**
```bash
vim resumes/base/resume-base.md
# 或
vim resumes/versions/<岗位>.md
```

2. **创建备份（重大修改时）**
```bash
TIMESTAMP=$(date +%Y-%m-%d-%H-%M)
cp resumes/base/resume-base.md resumes/base/resume-base-$TIMESTAMP.md
# 删除旧备份
rm -f resumes/base/resume-base-2026-*.md  # 保留最新版备份
```

3. **推送到 GitHub**
```bash
cd resumes
git add .
git commit -m "Update: 修改内容说明"
git push origin main
```

## 💡 注意事项

### 内容规范
- ✅ **简洁**: 只保留核心内容，删除冗余描述
- ✅ **量化**: 使用数据说明成果（如：成本降低 31%）
- ✅ **真实**: 所有经历和技能必须真实可靠
- ✅ **URL**: 使用完整地址，便于复制

### 板块顺序
- **严格遵守** 6 大板块顺序
- 新增岗位版本时保持一致
- 不要随意调整顺序

### 备份频率
- **重大修改**: 创建备份（如结构调整、岗位定制）
- **小修改**: 直接覆盖（如错别字、时间更新）
- **仅保留一份**: 删除旧备份，避免仓库膨胀

### 导出 PDF
- ❌ **仓库不存储** PDF 文件
- ✅ **自行导出**: 使用 Typora、VS Code 等工具
- ✅ **按需导出**: 投递时根据要求导出
