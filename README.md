## 🎯 岗位定制指南

### 1. 技术负责人 / 技术总监
**文件**: `versions/tech-lead.md`  
**侧重点**:
- 团队管理经验（外包管理、跨部门协作）
- 技术决策能力（技术选型、架构设计）
- 业务影响力（成本优化、效率提升）
- 数字化转型升级经验

**调整建议**:
- 突出软件部主管经历
- 强调从 0 到 1 搭建体系的能力
- 量化业务成果（成本降低 31%、交付周期缩短 30%）

### 2. AI 工程化 / 大模型应用

## ✏️ 创建新版本流程

1. **复制基础版本**:
```bash
cp base/resume-base.md versions/<岗位名称>.md
```

2. **根据岗位调整**:
- 调整"个人定位"描述
- 重新排序工作经历（相关经历前置）
- 删减不相关内容（保持 2 页以内）
- 强化岗位关键词

3. **检查要点**:
- [ ] 个人定位与岗位匹配
- [ ] 核心技能包含岗位 JD 关键词
- [ ] 工作经历按相关性排序
- [ ] 量化成果突出
- [ ] 无错别字和格式问题

## 💡 其他建议

### 1. 文件格式
- **Markdown**: 便于版本管理和快速调整
- **PDF**: 正式投递时使用（保持格式）
- **Word**: 部分公司要求可编辑版本

### 2. 命名规范
```
李越男_岗位_版本日期.md
李越男_技术负责人_20260311.md
```

### 3. 配套材料
- **求职信 (Cover Letter)**: 针对特别心仪的公司
- **项目集 (Portfolio)**: GitHub、博客链接
- **推荐信**: 前同事/上级的 LinkedIn 推荐
